# Point_Cloud_Registration
对于多种经典点云配准算法的实现以及提出的一种新的基于一点采样一致性点云配准算法的实现

这里对于ICP,RANSAC,FPFH,PointNet++这些经典点云配准算法进行复现

其中ICP通过找最近点的对应关系来计算最优变化，这是一个不停迭代直至收敛点过程
但是对于初始位姿较差，重叠率低的点云会产生大量错误对应，故一般只用于精配准阶段

FPFH通过引入特征描述子来极大提高了对应关键点的选择效率从而估算出一个较好的初始变换
RANSAC则是通过对关键点进行随机采样+投票的方法来排除错误的特征匹配
这两者一般共同作用于粗配准阶段

PointNet++引入了深度学习的方法，在PointNet的基础上增加了CNN思想使其对与复杂局部特征描述力更好，
同时利用全局特征（这里就相当于CNN的最后一层卷积能够获取全局特征）来快速实现初始变换，解决初始位姿的问题，
当然全局特征更多是用于场景识别以及快速判断多视角点云配准中哪些点云之间可以进行配准，
当然对于特定的配准更推荐重新训练权重以达到更好的效果。

至于我提出的CC-RANSAC则是利用构建局部参考轴以及一对关键点对来固定平移的3个自由度和旋转的2个自由度，
至于剩余的一个旋转自由度则是通过沿LRA轴方向生成从而确定得来。
